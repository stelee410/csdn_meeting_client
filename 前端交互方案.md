# CSDN 会议系统 - 前端交互方案

> 本文档描述后端提供的接口及前端交互方式

---

## 1. 通用规范

### 1.1 接口基地址

```
开发环境: http://localhost:8080
```

### 1.2 统一响应格式

所有接口返回统一的 JSON 格式：

```json
{
  "code": 200,        // 状态码，200 表示成功
  "msg": "success",   // 响应消息
  "data": {}          // 响应数据（可能为 null、对象或数组）
}
```

**状态码说明：**

| Code | 含义 | 处理建议 |
|------|------|---------|
| 200 | 成功 | 正常处理数据 |
| 400 | 请求参数错误 | 提示用户检查输入 |
| 401 | 未登录或登录已过期 | 跳转登录页 |
| 403 | 权限不足 | 提示用户无权限 |
| 422 | 业务逻辑错误 | 显示 msg 错误信息 |
| 500 | 服务器内部错误 | 稍后重试或联系管理员 |

### 1.3 认证方式

使用 **JWT Token** 认证，Token 有效期 2 小时。

**登录后请求头格式：**

```
Authorization: Bearer <accessToken>
```

## 2. 认证接口

### 2.1 密码登录

**接口信息**

- **URL**: `/api/auth/login/password`
- **Method**: POST
- **Content-Type**: `application/json`
- **无需认证**: 是

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mobile | string | 是 | 手机号，11位数字 |
| password | string | 是 | 明文密码（传输依赖 HTTPS） |

**请求示例**

```json
{
  "mobile": "13800138000",
  "password": "password123"
}
```

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "userInfo": {
      "userId": "U123456",
      "mobile": "138****8000",
      "nickname": "张三",
      "avatarUrl": "https://example.com/avatar.jpg",
      "userType": "USER",
      "email": "zhangsan@example.com",
      "emailVerified": true,
      "realName": "张三",
      "company": "CSDN",
      "jobTitle": "高级工程师",
      "industry": "互联网",
      "status": "NORMAL",
      "agreementAccepted": true,
      "privacyAccepted": true,
      "createdAt": "2024-01-15T10:30:00",
      "lastLoginAt": "2024-03-29T14:20:00"
    }
  }
}
```

**错误响应**

```json
// 手机号未注册
{
  "code": 422,
  "msg": "手机号未注册"
}

// 账号被冻结
{
  "code": 422,
  "msg": "账号已被冻结，请联系客服"
}

// 密码错误
{
  "code": 422,
  "msg": "手机号或密码错误"
}
```

### 2.3 发送短信验证码

**接口信息**

- **URL**: `/api/auth/verification-code/sms`
- **Method**: POST
- **Content-Type**: `application/json`
- **无需认证**: 是

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| target | string | 是 | 手机号 |
| scene | string | 是 | 场景：REGISTER(注册)、LOGIN(登录)、RESET_PASSWORD(重置密码) |

**请求示例**

```json
{
  "target": "13800138000",
  "scene": "LOGIN"
}
```

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

**错误响应**

```json
// 发送过于频繁
{
  "code": 422,
  "msg": "验证码发送过于频繁，请60秒后重试"
}
```

### 2.4 用户注册

**接口信息**

- **URL**: `/api/auth/register`
- **Method**: POST
- **Content-Type**: `application/json`
- **无需认证**: 是

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mobile | string | 是 | 手机号 |
| smsCode | string | 是 | 短信验证码 |
| password | string | 是 | 密码（至少8位，包含字母和数字） |
| confirmPassword | string | 是 | 确认密码 |
| agreementAccepted | boolean | 是 | 是否同意用户协议 |
| privacyAccepted | boolean | 是 | 是否同意隐私政策 |
| nickname | string | 否 | 昵称 |
| avatarUrl | string | 否 | 头像URL |
| email | string | 否 | 邮箱（填写则需同时提供 emailCode） |
| emailCode | string | 否 | 邮箱验证码 |

**请求示例**

```json
{
  "mobile": "13800138000",
  "smsCode": "123456",
  "password": "password123",
  "confirmPassword": "password123",
  "agreementAccepted": true,
  "privacyAccepted": true,
  "nickname": "张三"
}
```

**响应数据**：同登录成功响应，注册成功后自动登录

### 2.6 退出登录

**接口信息**

- **URL**: `/api/auth/logout`
- **Method**: POST
- **Content-Type**: `application/json`
- **无需认证**: 是（前端清除 token 即可）

**请求示例**

```json
{}
```

**处理建议**

前端应同时执行：
1. 调用此接口（可选，用于服务端记录）
2. 清除本地存储的 `accessToken` 和 `userInfo`
3. 跳转登录页面

## 5. 前端交互流程

### 5.1 登录流程

```
用户输入账号密码
    ↓
调用 POST /api/auth/login/password
    ↓
├─ 成功：保存 token 和用户信息，跳转目标页
└─ 失败：显示错误提示（根据 code 和 msg）
```

### 5.2 短信登录流程

```
用户输入手机号
    ↓
点击获取验证码 → 调用 POST /api/auth/verification-code/sms
    ↓
倒计时60秒，等待用户输入验证码
    ↓
用户输入验证码 → 调用 POST /api/auth/login/sms
    ↓
├─ 成功：保存 token 和用户信息，跳转目标页
└─ 失败：显示错误提示
```

### 5.3 Token 处理

**存储位置**：
- `localStorage` 或 `sessionStorage` 存储 `accessToken`
- 建议同时存储 `userInfo` 用于页面展示

**请求拦截**：
- 所有需要认证的接口自动添加请求头：`Authorization: Bearer <token>`

**响应处理**：
- 收到 401 状态码时，清除本地 token，跳转登录页

### 5.4 TODO【需与CSDN对接】CSDN 扫码登录流程

```
用户点击"CSDN扫码登录"
    ↓
前端展示二维码（TODO【需与CSDN对接】调用 CSDN 接口获取）
    ↓
TODO【需与CSDN对接】用户用 CSDN App 扫码确认
    ↓
TODO【需与CSDN对接】CSDN 回调返回 authCode
    ↓
前端调用 POST /api/auth/csdn/callback
    ↓
├─ 已注册用户：直接登录成功
└─ 新用户：跳转补充资料页（设置密码、同意协议）
        ↓
    用户填写后再次调用 /api/auth/csdn/callback
        ↓
    注册并登录成功
```

## 6. 错误处理建议

### 6.1 通用错误提示

| 场景 | 提示方式 |
|------|---------|
| 参数校验错误（400） | 在对应输入框下方显示错误信息 |
| 业务逻辑错误（422） | Toast 或弹窗显示 msg 内容 |
| 未登录（401） | 跳转登录页，登录后返回原页面 |
| 服务器错误（500） | 友好提示"服务器繁忙，请稍后重试" |

### 6.2 登录相关错误

| 错误信息 | 前端处理 |
|---------|---------|
| 手机号未注册 | 提示"该手机号未注册，是否前往注册？" |
| 手机号或密码错误 | 提示"手机号或密码错误"（不明确提示哪个错） |
| 验证码错误或已过期 | 提示"验证码错误，请重新获取" |
| 账号已被冻结 | 提示"账号已被冻结，请联系客服" |
| 密码强度不足 | 提示"密码需至少8位，包含字母和数字" |

## 7. 密码安全说明

### 7.1 传输安全

- 密码以**明文形式**通过 HTTPS 传输
- 依赖 HTTPS 协议保证传输安全
- 建议生产环境强制使用 HTTPS

### 7.2 存储安全

- 数据库使用 **BCrypt** 算法加密存储密码
- 密码不可逆，即使数据库泄露也无法还原明文
