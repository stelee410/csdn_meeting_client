# 🛠️ 后端开发内容说明：发起会议、我的会议与商业化管理

本项目核心目标是将 CSDN 的会议服务从"人工销售模式"转变为"自助产品模式"。作为后端开发，你负责构建办会方（企业或个人）从**创建会议**到**数据复盘**，再到**付费推广**的全链路支撑服务。

---

## 1. 会议创建与智能化管理

### 1.1 三种创建方式

* **AI 辅助填单**：对接大模型 API，支持上传 Word/PDF（<20MB）或图片（<10MB，JPG/PNG），系统自动解析并提取会议标题、时间、地点、简介、主办方等字段回填表单。解析须在 **15 秒**内完成，超时则降级为手动模式（返回 422）。上传文件需进行病毒扫描；AI 回填内容需经敏感词过滤，含敏感词时标红提示。
* **活动模板**：维护预置的 5-10 个技术活动模板（如"技术沙龙"、"新品发布会"、"黑客马拉松"、"技术公开课"），办会方可浏览、预览并一键复用（模板含预置场景、简介骨架、默认人群标签）。模板本身由运营端维护（增删改），后端提供管理员权限的 CRUD 接口。
* **空白创建**：直接进入空白表单，仅需填写会议标题即可保存草稿。

### 1.2 基础字段与联动逻辑

后端需对以下核心字段进行存储与校验：

| 字段 | 必填 | 说明 |
|------|------|------|
| 会议名称 | 是 | 最大 50 字；AI 自动填充 |
| 主办方（公司） | 是 | 默认企业账号认证名，可修改 |
| 会议形式 | 是 | 线上/线下/结合；选「线下」或「结合」时举办地址必填 |
| 会议场景 | 是 | 如"开发者会议"、"区域营销"等；选"区域营销"时涉及区域必填 |
| 会议时间 | 是 | 精确到分钟；AI 自动填充 |
| 举办地域 | 条件 | 会议形式含"线下"时必填（省市级联多选） |
| 会议标签 | 是 | 最多 5 个；支持 AI NLP 推荐 + 用户手动输入 |
| 适合人群 | 是 | 职级（初/中/高）+ 技术方向（Java/AI/前端等）多选；用于精准推荐 |
| 会议简介 | 是 | 富文本支持图文混排；AI 自动填充 |
| 会议海报 | 是 | 封面图，16:9 比例 |

**保存草稿**：仅校验"会议名称"必填，其他字段可为空，状态为 DRAFT，不触发审核。提交审核时进行全表单校验（必填项 + 格式 + 敏感词初步拦截）。

### 1.3 智能标签系统

用户填写完"会议标题"和"会议简介"后（光标离开输入框时），后端异步调用 NLP 接口，分析文本内容并返回 3-5 个推荐标签（Tag）。推荐标签供用户确认选择，亦可手动输入新标签。最终保存的标签（系统推荐 + 用户手动确认）存入标签表，最多 5 个，用于算法推荐与检索。

### 1.4 会议日程与分会场（四级配置）

后端需支持「会议日 → 环节 → 分会场 → 议题」四级树状结构：

* **会议日（ScheduleDay）**：日期必须在会议开始/结束日期范围内；同一会议下日期不重复。
* **环节（Session）**：开始/结束时间必须落在该日内（不跨日）；同一日内环节时间不得重叠。
* **分会场（SubVenue）**：同一环节内可并行多个分会场，名称必填（如"主会场"、"JAVA 分会场"）。
* **议题（Topic）**：主题名称必填（最大 100 字）；嘉宾列表选填（可关联专家画像或手动输入）；可选：议题简介、所涉及产品。

**提交审核时**的四级结构完整性校验规则：至少 1 个日程日 → 每日至少 1 个环节 → 每个环节至少 1 个分会场 → 每个分会场在该环节下至少 1 个议题（主题名称必填）。

简单场景（半天单会场）：仅配置 1 个日程日、1 个环节、1 个分会场、1 个议题即可，无需多会场配置。

## 2. 会议状态机（完整版）

后端必须严格实现以下完整流转，禁止跳变：

| 触发操作 | 来源状态 | 目标状态 | 触发方 | 备注 |
|---------|---------|---------|------|------|
| 保存草稿 | 无 | DRAFT | 办会方 | 仅校验标题 |
| 提交审核 | DRAFT / REJECTED | PENDING_REVIEW | 办会方 | 校验四级日程完整性 |
| 撤回审核 | PENDING_REVIEW | DRAFT | 办会方 | 恢复编辑权限 |
| 审核通过 | PENDING_REVIEW | PUBLISHED | 管理员 | 审计日志必记 |
| 审核拒绝 | PENDING_REVIEW | REJECTED | 管理员 | 需记录拒绝原因；审计日志必记 |
| 再次提交 | REJECTED | PENDING_REVIEW | 办会方 | 可修改信息后重提 |
| 自动开始 | PUBLISHED | IN_PROGRESS | 定时任务 | 按 startTime 触发 |
| 自动结束 | IN_PROGRESS | ENDED | 定时任务 | 按 endTime 触发；触发后启动简报聚合 |
| 主动下架 | PUBLISHED / IN_PROGRESS | OFFLINE | 办会方 | 需填写下架原因；审计日志必记 |
| 强制下架 | 任意 | OFFLINE | 管理员 | 审计日志必记 |
| 逻辑删除 | DRAFT / ENDED / OFFLINE / REJECTED | DELETED | 办会方 | 软删除，status = DELETED |

## 3. 我的会议（三页签）

### 3.1 我报名的会议

* 数据范围：当前用户有有效报名记录的会议。
* **默认**仅展示状态为「已发布」「进行中」的会议；支持"包含已结束"筛选。
* 排序：按会议开始日期倒序（日期最近的在上方，日期较远的在下方）。
* 空状态引导：无数据时提示"暂无报名的会议"并引导至会议列表。

### 3.2 我收藏的会议

* 复用 CSDN 现有收藏体系，`收藏类型 = 会议`，关联 `meeting_id`；若现有体系不支持会议类型，需扩展或新建关联表。
* 列表至少展示会议名称；点击跳转会议详情，详情页收藏按钮呈已激活状态。
* 取消收藏后，「我收藏的会议」列表同步更新（该会议消失）。

### 3.3 我创建的会议（列表与状态管理）

* 支持按状态（全部 / 新建 / 待审核 / 已发布 / 进行中 / 已结束 / 已下架 / 已拒绝）、时间范围筛选。
* 列表项展示：封面、标题、时间、状态、核心数据（浏览量/报名数）、操作列。
* 状态操作接口：提交审核、撤回审核、下架（需填原因）、逻辑删除（`status = DELETED`）。
* 已发布状态的会议在操作列额外提供：「审核报名」入口、会议简报入口、数据统计入口。
* 正在推广中的会议需展示推广状态标识。

---

## 4. 数据统计与高阶权益

### 4.1 基础数据（全量免费）

实时提供：浏览量（PV）、访客数（UV）、报名人数、签到人数、签到率；支持近 7 日趋势图展示。

### 4.2 会议数据高阶权益包

* 以**具体会议**为粒度购买，会议发布后即可购买（无需等待会议结束）。
* 解锁后可查看已报名用户的整体画像聚合：公司类型分布、岗位占比、职级分布（初/中/高）、行业分布、技术栈词云。
* **仅提供整体聚合分析**，不暴露任何具体参会者的个人明细信息。
* 权益价格：由运营后台配置，或通过预留的后端/运维配置接口下发（前期无法实现运营配置时预留接口）。
* 购买后，该会议详情中的用户画像数据与会议简报的高阶数据**同时解锁**（同一权益包）。
* 购买流程：调用 CSDN 统一收银台完成支付 → 系统记录 `MeetingRights`（DATA_PREMIUM） → 同步更新 `Meeting.isPremium = true`。

## 5. 会议简报

* 触发时机：会议进入 ENDED 状态后由系统自动触发数据聚合；也支持办会方主动点击"下载简报"。
* **基础内容**（所有人）：基础统计数据（PV/UV/报名数/签到数）、热门议题回顾（基于议题点击/评分）。
* **高阶内容**（已购买权益）：参会者整体画像分析（公司/职级/行业/技术栈等聚合结果），仅整体分析，不展示单个参会者信息。未购买时该章节显示占位提示。
* 输出格式：PDF（直接打印/汇报）、Word（二次编辑），浏览器触发文件下载。

## 6. 推广配置

仅「已发布」状态的会议可配置推广；「待审核」「新建」等状态下，配置推广按钮置灰。需对接 CSDN 广告系统。核心配置包含以下 5 个模块：

1. **产品信息**：产品名称（默认填充会议名称）、用户意图（多选：知识点问答/代码报错/开发工具/体系学习/考试求职/行业资讯等）、精准投放等级（1-4级）、扩展产品词。
2. **用户行为圈选**：行为周期（7/15/30/60/90 天）、行为类型（搜索/创作）、用户行为频次、近期触达（近半年点击过短信/邮件）。
3. **静态信息圈选**：地域（省市级联精确到市级）、学历、毕业院校、是否学生、行业领域（AI/云计算/游戏/金融/教育等）、公司名称、职位（CTO/架构师/高级/中级/初级工程师等）、其他标签。
4. **渠道选择**：短信、邮件、私信、Push（多选）。
5. **付费形式**：CPM（按曝光）、CPC（按点击）、CPA（按报名），可多选。

后端实时计算预估效果（调用 CSDN 大数据用户画像库接口）：预计覆盖人数、预计曝光、预计点击、总价。

生成推广订单后：① 通知管理后台留档（便于运营财务跟进）；② 30 分钟内完成支付享 85 折优惠。

> 账单列表与发票申请前端功能**本期不提供**，但后端需建立 `t_meeting_bill` 表记录所有费用流水。

## 7. 参会人审核（报名管理）

* 仅「已发布」状态的会议支持参会人审核入口（已结束/已下架/已拒绝/新建状态不展示）。
* **双入口**：「我创建的会议」操作列「审核报名」按钮 + 会议详情页「报名管理」入口，进入同一审核页面。
* 审核操作：
  * **通过** → 报名记录 `status = APPROVED`；系统向报名人发送多渠道通知（Push、短信、邮件、私信/站内信）。
  * **拒绝** → 报名记录 `status = REJECTED`；系统向报名人发送审核拒绝通知（同渠道）。
* 通知文案含会议名称和审核结果，由运营/产品确定具体模板。

---

## 8. 开发边界与约束

* **复用体系**：直接复用 CSDN 既有账号与登录态，不开发独立账户系统；收藏功能复用现有收藏服务，无需重建。
* **账单前端**：本期不提供账单列表与发票申请前端功能，仅建立后端数据结构（`t_meeting_bill`）记录费用流水。
* **运营审核端 UI**：审核管理后台 UI 不在本模块范围内，仅提供状态流转接口。
* **移动端差异**：移动端仅提供参会和基础管理接口，管理员级别的审核逻辑仅在网页端后端生效。
* **不含线下流转**：不涉及线下纸质合同、实物发票寄送等传统商务流程。
* **长图分享**：V2 规划功能（推广长图/二维码），本期仅预留接口，不实现。
